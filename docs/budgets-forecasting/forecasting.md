# Forecasting

## Overview

The **Forecast** module helps organizations proactively manage cloud budgets by tracking allocated budgets, forecasts, and actual spend across projects. It provides detailed, month-wise breakdowns and allows users to request and approve forecast changes.

## Forecast Summary Cards

At the top of the Forecast page, high-level budget metrics:

- **Total Allocated Budget:** Total budget assigned across all projects
- **Total Budget Spend:** Actual cloud spend to date
- **Total Forecast:** Sum of forecasted costs for all projects
- **Total Projected:** Projected spend
- **Requested Forecast Change:** Total forecast change under review
- **Variance:** Difference between budget and actual spend

## Forecast Table Overview

Lists all active projects with:
- Project Name
- Budget (Yr)
- Forecast (Yr)
- Forecast Change status
- Start Date
- Actions (View, Edit)

## Viewing Forecast Details

Click ellipsis (···) → **View Forecast** to see:
- Budget per month
- Spend per month
- Forecast per month
- Variance (positive in green, negative in red)
- Requested Change
- New Forecast ($) after approval

## Editing Forecasts

**Project Admins** can submit forecast changes:
1. Edit "New Forecast ($)" field
2. Provide reason for change
3. Click **Save Forecast**
4. Change request sent for approval

## Forecast Approval Workflow

1. Workspace Admins/Users receive notification
2. Review request
3. **Approve** or **Reject** with comment
4. Once approved, forecast updates
5. Reflected in tables and summary metrics

## Change Logs Tab

Records history of all forecast changes:
- Who made changes
- When changes occurred
- What was modified
- Approval status
