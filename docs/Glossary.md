# Glossary

## Billing View
A saved cost view that defines filters, grouping, and metrics for analysis.

## Cost Assignment Rule
Rules that allocate spend to projects using account, region, and tag criteria.

## Cost Types
Different cost perspectives such as billed, effective, and list cost.

## MSA (Master Service Account)
An organization-scope service account used like a PSA but across all projects.

## PSA (Project Service Account)
A project-scope service account used to access cloud provider data for a specific project.

## Policy
A governance or optimization rule that evaluates cloud resources and triggers actions.

## Project
A logical grouping of cloud resources based on accounts, subscriptions, or tags.

## Project Group
A higher-level grouping of projects for roll-up reporting and governance.

## Task
An actionable item generated from policies, recommendations, or user requests.

## Ticket
A work item synced with external systems such as Jira or ServiceNow.

## Workflow
A set of rules and actions that route policy findings to tasks, tickets, or automations.

## Workspace
An organizational boundary for data, access, and governance.
